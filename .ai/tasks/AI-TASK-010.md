# AI-TASK-010 — 빈 코스 표시, 코스 상세 편집/장소추가/친구 공유 흐름 개선

## 요청 요약

코스를 만든 직후 장소가 없는데 예시 장소가 들어간 것처럼 보이는 문제를 고치고, 코스 상세 화면의 불필요한 버튼을 제거합니다. 편집 버튼으로 제목/날짜를 수정할 수 있게 하고, 장소 추가는 지도 탭 이동이 아니라 코스 상세 안에서 지도형 검색/선택 오버레이로 처리합니다. 친구 공유는 서비스 내 친구에게 요청을 보내고, 친구가 알림에서 수락해야 보기 전용으로 볼 수 있는 흐름으로 정리합니다.

## 조사 기준

- 빈 코스가 가짜 장소처럼 보이는 원인:
  - `CourseDetailPage.tsx`에서 저장/서버 코스가 없을 때 `defaultStops`를 렌더링
  - `CourseDiscoveryCard.tsx`에서 서버 코스의 items가 비어도 `첫 장소`, `다음 장소`, `마무리 산책`을 표시
- 코스 상세 상단에 다운로드/지도/친구 아이콘이 있고, 아래에 친구 공유 버튼이 중복 존재
- `교통`, `예산`, `체크리스트` 버튼은 현재 실제 기능 없이 UI만 노출
- 기존 장소 추가는 `/map?mode=course-add`로 이동해 지도 페이지에서 append 수행
- 친구 공유 바텀시트는 하드코딩 배열을 사용하고 있으며 실제 친구 목록 API는 `getFriends()`가 별도 존재

## 구현 계획

1. 빈 코스일 때 가짜 장소를 표시하지 않고 카드/상세 모두 빈 상태로 표시한다.
2. 코스 상세 상단의 친구 아이콘과 `교통`, `예산`, `체크리스트` 버튼을 제거한다.
3. 편집 바텀시트를 추가해 제목과 날짜를 수정하고, 서버 코스는 `updateCourse`, 로컬 코스는 storage 업데이트 함수로 저장한다.
4. 장소 추가 버튼은 상세 내부 지도 선택 오버레이를 열도록 변경한다.
5. 지도 선택 오버레이에서 검색 결과/지도 핀 중 하나를 체크로 선택해야만 코스 맨 뒤에 추가한다.
6. 친구 공유는 실제 친구 목록만 표시하고, 선택 시 “공유 요청” 상태로 처리한다. 친구가 알림에서 수락해야 보기 전용 접근이 생긴다는 문구와 상태를 반영한다.
7. 빌드 검증을 실행하고 작업 로그를 남긴다.

## 위험 및 대응

- **지도 UI 재사용 범위:** 새 컴포넌트 파일 추가 없이 상세 파일 내부에서 기존 지도 hook/API를 사용해 구현한다.
- **친구 공유 API 부재:** 현재 코스 공유 요청 전용 API가 없으므로 프론트 상태와 안내 문구를 먼저 맞춘다. 서버 API가 추가되면 요청 mutation만 연결하면 되도록 구조를 둔다.
- **로컬/서버 코스 저장 방식 차이:** 로컬 코스는 storage 업데이트 함수를 추가하고, 서버 코스는 기존 `updateCourse`를 재사용한다.
- **빈 코스 지도 처리:** 경로 지도는 장소가 없을 때 중심 계산을 하지 않고 빈 상태를 렌더링한다.
